7. What Is the Difference Between the ‘id’ Attribute and the ‘class’ Attribute of HTML Elements?
Ans: 1. id Attribute: Used to uniquely identify a single element on a page. The id value must be unique within the HTML document.
     2. class Attribute: Used to assign one or more classes to multiple elements. Multiple elements can share the same class name.

11. What Is the Difference Between a Block-Level Element and an Inline Element?
Ans:  1. Block-Level Element: Takes up the full width available and starts on a new line.
           Examples include <div>, <p>, <h1>, <ul>, and <table>.

      2. Inline Element: Only takes up as much width as necessary and does not start on a new line.
           Examples include <span>, <a>, <img>, <strong>, and <em>.

16. How Are Active Links Different from Normal Links?
Ans:  1. Normal Links: Links that have not been clicked by the user. Typically displayed in blue by default.
      2. Active Links: Links that are currently being clicked or are in focus. They are often styled differently 
                       (e.g., changing color) to indicate their active state.

17. What Are the Different Tags to Separate Sections of Text?
Ans: 1. <div>: Used to group block-level content.
     2. <p>: Defines a paragraph.
     3. <hr>: Inserts a horizontal line (thematic break).
     4. <br>: Inserts a line break.
     5. <section>: Defines a section in a document.
     6. <article>: Defines an independent, self-contained content.
     7. <header>, <footer>, <aside>: Defines different parts of the document structure.

18. What Is SVG?
Ans: SVG (Scalable Vector Graphics) is an XML-based format for creating two-dimensional vector images.
     It allows for scalable, high-quality graphics that do not lose clarity when resized.

19. What Is the Difference Between HTML and XHTML?
Ans:  1. HTML (HyperText Markup Language): The standard markup language for creating web pages. It is more lenient with syntax rules.
      2. XHTML (eXtensible HyperText Markup Language): A stricter, XML-based version of HTML. It enforces strict syntax rules

20. What Are Logical and Physical Tags in HTML?
Ans:  1. Logical Tags: Describe the content and its structure. They do not define the appearance but rather the role of the 
         content.
         examples  include     <em>,  <strong>, <cite>, <abbr>, <code>.

      2.Physical Tags: Directly affect the appearance of the content. They define how the text should look.
         Examples include <b> (bold),    <i> (italic), <u> (underline), <sub> (subscript), <sup> (superscript). However,
              with the separation of content and style in modern web development, logical tags are preferred over physical tags.
